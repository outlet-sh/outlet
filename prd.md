## 7. Backup System Architecture

### Why This Matters

SQLite's single-file database is a feature, not a limitation. One file = one backup = complete data portability. Users' email lists are their most valuable assetâ€”backup must be trivial.

### Dashboard-First Configuration

All backup settings configured in **Settings â†’ Backup** (no YAML required):

**Manual Backup:**
- "Download Backup" button â†’ browser downloads `outlet-backup-YYYY-MM-DD.db`
- "Backup Now" button â†’ immediate S3 backup (if configured)

**S3 Automated Backup:**
- Toggle: Enable S3 Backup
- Bucket name
- Region (dropdown)
- Access Key ID
- Secret Access Key (encrypted at rest)
- Schedule: Daily / Every 12 hours / Every 6 hours / Hourly
- Retention: X days (auto-cleanup of old backups)
- "Test Connection" button (validates before saving)

**Backup History Table:**
- Timestamp, size, location (local/S3), status
- One-click restore button per backup
- One-click download button for S3 backups

### Technical Implementation

- `robfig/cron` for scheduling (goroutine, no external dependencies)
- SQLite Online Backup API (safe during live writes)
- Credentials encrypted in SQLite using same pattern as SES keys
- Failure notifications via configured email address

### Restore Process

**From Dashboard:**
- Click restore button next to any backup in history
- Or upload a backup file directly

**Manual:**
1. Stop Outlet.sh
2. Replace `outlet.db` with backup file
3. Start Outlet.sh
4. Goose verifies/runs any needed migrations
5. Done

### Backup Contents

The single `outlet.db` file contains everything:
- All subscribers and custom fields
- All lists and tags
- All campaigns and sequences
- All templates (marketing + transactional)
- All analytics/tracking data
- All automation rules
- All settings (including encrypted S3/SES credentials)

## 8. Deployment

### Option 1: Direct Binary

**Download and run:**
```bash
wget https://outlet.sh/download/outlet-linux-amd64
chmod +x outlet-linux-amd64
./outlet-linux-amd64
```

**Systemd service for auto-restart:**

Create `/etc/systemd/system/outlet.service`:
```ini
[Unit]
Description=Outlet.sh Email Platform
After=network.target

[Service]
Type=simple
User=outlet
WorkingDirectory=/opt/outlet
ExecStart=/opt/outlet/outlet
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl enable outlet
sudo systemctl start outlet
```

### Option 2: Docker Compose

Create `docker-compose.yml`:
```yaml
services:
  outlet:
    image: outlet/outlet:latest
    ports:
      - "443:443"
      - "80:80"
    volumes:
      - ./data:/data
    restart: always
```

```bash
docker compose up -d
```

**That's it.** Data persists in `./data/outlet.db`. Backups, updates, configâ€”all in the dashboard.
